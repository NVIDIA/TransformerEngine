Motivation

Why Not Just Use Hugging Face Directly?

HF Mixtral Limitations

  • Serial expert dispatch: Python for loop over 8 experts launches 16 small GEMMs per layer, each too small to saturate tensor cores
  • No expert parallelism: Only pipeline parallelism via device_map="auto" — layers split across GPUs, but all 8 experts live on the same GPU
  • No FP8 support: Stuck at BF16/FP16, cannot leverage Blackwell's doubled FP8 tensor-core throughput
  • Sequential GPU usage: Only 1 GPU active at a time (7/8 idle in pipeline parallel)

HF Expert Loop (16 GEMMs/layer)

for expert_idx, expert_layer in enumerate(self.experts):
    idx, top_x = torch.where(expert_mask[expert_idx])
    current_state = hidden_states[None, top_x]
                    .reshape(-1, hidden_dim)
    current_hidden = expert_layer(current_state)
                     * routing_weights[top_x, idx, None]
    final_hidden_states.index_add_(
        0, top_x, current_hidden
    )

8 experts × (w1/w3 + w2) = 16 kernel launches per MoE layer
32 layers × 16 = 512 small GEMMs per forward pass

TE: 1 Kernel Launch

self_attention.layernorm_qkv.weight
  # [6144, 4096]  -- Q+K+V fused
self_attention.layernorm_qkv.layer_norm_weight
  # [4096]  -- fused into same module

# LayerNorm + QKV GEMM = 1 fused kernel

Interleaved GQA Packing

Mixtral uses GQA: 32 Q heads, 8 KV heads (4 Q per group):

# Total QKV dim = 32 + 2*8 = 48 slices
# Each slice is [head_size=128, hidden=4096]

fused_qkv.view(48, 128, 4096)

# For KV group i (i = 0..7):
#   Q heads: indices (6*i) to (6*i + 3)
#   K head:  index   (6*i + 4)
#   V head:  index   (6*i + 5)

# Copy Q into fused layout:
for i in range(8):  # 8 KV groups
    start = (4 + 2) * i
    fused[start : start+4] = q[i*4 : (i+1)*4]
    fused[start + 4]       = k[i]
    fused[start + 5]       = v[i]

Section 2 — Expert Parallelism

Expert Parallelism: All-to-All Token Dispatch

EP=8 Architecture

  • Each GPU holds 1 expert per layer (8 experts / 8 GPUs)
  • Non-expert params (attention, norms, embeddings, router) are replicated
  • Expert weights stored as DTensor(Shard(0))
  • Each rank processes its own data shard (EP = DP for data)

Dispatch Cycle (4 NCCL ops/layer)

Router

→

→

→

→

→

→

4 NCCL ops × 32 layers = 128 collectives per step

Setup

config.expert_parallel_size = 8
ep_mesh = DeviceMesh("cuda", torch.arange(8))
model.model.set_ep_groups(
    ep_group=dist.group.WORLD,
    ep_mesh=ep_mesh
)
# Expert weights: DTensor(Shard(0))
# Global: [8, 28672, 4096]
# Local:  [1, 28672, 4096] per rank

Differentiable All-to-All

class _DifferentiableAllToAll(autograd.Function):
  @staticmethod
  def forward(ctx, input, out_splits,
              in_splits, group):
    output = torch.empty(sum(out_splits), ...)
    dist.all_to_all_single(
        output, input,
        out_splits, in_splits, group=group)
    return output

  @staticmethod
  def backward(ctx, grad_output):
    # Reverse: swap in/out splits
    return grad_input, None, None, None

Section 3 — Fused Token Dispatch

FusedTokenRouter via DeepEP

AllToAllTokenDispatcher (Tier 2)

1. moe_permute(index)
sort tokens by expert
2. all_to_all (counts)
exchange per-expert token counts
3. all_to_all (tokens)
exchange token data
4. _build_expert_sort_indices
re-sort for GroupedLinear

4 separate operations, serialized on one CUDA stream.

FusedTokenRouter (Tier 3)

1. fused_dispatch (DeepEP)
comm + permute fused
2. fused_indices_to_multihot
Triton kernel
3. moe_permute(mask)
sort for GroupedLinear

Key advantages:

  • Token deduplication: each token sent once per destination GPU, not once per expert
  • Fused comm+permute: DeepEP Buffer.dispatch() handles layout + transfer in one kernel
  • Triton index conversion: sparse [N, top_k] → dense [N, num_local_experts] multihot mask
recv_x, recv_indices, recv_probs, tokens_per_expert, handle = fused_dispatch(
    hidden_states, selected_experts, routing_weights.float(), num_experts, ep_group)
multihot_mask, probs = fused_indices_to_multihot(recv_indices, recv_probs, num_local_experts)
permuted_x, row_id_map = moe_permute(recv_x, multihot_mask.int(), num_out_tokens, map_type="mask")

One-Line Recipe Change

# Old: per-tensor delayed scaling (Hopper)
fp8_recipe = te_recipe.DelayedScaling()

# New: block scaling (Blackwell-native)
fp8_recipe = te_recipe.MXFP8BlockScaling()

# Applied via model constructor:
config.layer_precision = ["fp8"] * num_layers
model = NVMixtralForCausalLM(
    config, fp8_recipe=fp8_recipe)

Alignment Padding Required

MXFP8 requires both dims divisible by 32. Data-dependent token counts after all-to-all or THD unpadding need padding:

remainder = num_tokens % 32
if remainder != 0:
    pad = 32 - remainder
    tokens = F.pad(tokens, (0, 0, 0, pad))
    # ... run GEMM ...
    output = output[:orig_tokens]

Throughput: The Real Story

The Apples-to-Oranges Problem

  • HF (1 process): 1 × 2 × 8192 = 16K tokens/step
  • TE EP (8 ranks): 8 × 2 × 8192 = 128K tokens/step

Accelerate's BatchSamplerShard gives each EP rank a different data shard. TE processes 8x more unique data per step.

Summary

Takeaways

What Transformer Engine Provides

  • Fused kernels — GroupedLinear batches 8 expert GEMMs into 2 launches; LayerNormLinear fuses norm + QKV
  • Expert Parallelism — Distributes experts across GPUs with differentiable all-to-all and DTensor checkpoint support
  • Fused dispatch — DeepEP reduces communication overhead with token deduplication
  • MXFP8 — Blackwell-native block scaling: 1.73x speedup, 2x max batch size

Migration Effort

  • Weight mapping: replace_params() — one-time, ~200 LOC
  • Expert Parallelism: expert_parallel_size = 8 + set_ep_groups()
  • Fused dispatch: dispatcher_type = "fused"
  • MXFP8: fp8_recipe = MXFP8BlockScaling()
  • No training loop changes for any tier

Open Items

  • Gradient allreduce for replicated params (EP without DDP)
  • MXFP8 dim-32 alignment padding for variable token counts
